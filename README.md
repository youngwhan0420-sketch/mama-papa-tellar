1. 주제 선정 및 문제 탐색<br>
   • 주제:부모의 목소리를 복제하여 감정 풍부하게 읽어주는 태블릿 기반 인터랙티브 구연동화 서비스<br>
   • 배경:맞벌이 가구의 증가로 부모가 아이에게 책을 읽어줄 물리적 시간이 부족해지고 있습니다.<br>
   아마존 알렉사(Alexa)나 네이버 클로바(Clova) 등 글로벌 기업들도 '보이스 클로닝'을 통한 정서적 유대감을 강조하고 있습니다.<br>
   본 팀은 단순히 글자를 읽는 것을 넘어, 부모의 음색과 동화 속 감정을 결합해 아이에게 최고의 몰입감을 선사하고자 합니다.

2. 사용자 분석<br>
   • 사용자 특성:6세 아들을 둔 30대 맞벌이 직장인 (워킹맘/워킹대디)<br>
   • 사용 상황:퇴근 후 지친 몸으로 아이의 잠자기 전 구연동화 요청을 들어줘야 하는 상황<br>
   • 불편함/문제 상황:직접 읽어주기엔 체력이 고갈되었고, 일반 TTS 앱은 기계적인 말투 때문에 아이가 금방 흥미를 잃고 부모의 목소리를 그리워함.<br>
   • 기존 방식의 한계:유튜브나 기존 동화 앱은 일방적인 시청각 자극만 제공하며, 아이의 돌발 질문에 대처하거나 부모 특유의 다정한 어감을 재현하지 못함.

3. 문제 정의<br>
   "맞벌이 부모 사용자는 기존 동화 앱의 단조로운 기계음과 일방적 전달 방식으로 인해 아이와의 정서적 교감 및 몰입도 저하라는 불편을 겪고 있으므로, 이를 해결할 수 있는 부모 음성 복제 및 감정 기반 인터랙티브 구연동화 서비스가 필요하다.“

5. 솔루션 기획
   • 서비스명:맘스텔러 (Mom’s Teller)
   • 서비스 목적:부모의 음색을 복제하고 동화 문맥에 맞는 감정을 입혀 육아 부담을 줄이고 아이의 정서 발달을 돕는 인터랙티브 서비스 제공.
   • 핵심 기술 활용 이유:
   • STT (Whisper):아이의 불분명한 발음을 정확히 캐치하여 대화의 흐름을 유지.
   • LLM (Llama 3/OpenAI):동화의 문맥을 '이해'하고 상황에 맞는 감정 태그와 답변을 생성하는 컨트롤 타워 역할.
   • TTS (ElevenLabs):1분의 녹음으로 부모의 개성을 재현하며, LLM의 지시에 따라 실시간으로 감정 톤을 변경.

[유형 2] 기반 모델 구현 상세 및 6인 역할 분담
각 단계는 [유형 2]의 표준 공정을 따르며, 초급 팀이 3주 안에 완수할 수 있도록 실질적인 업무로 쪼개었습니다.

1. 보이스 AI 엔진 (TTS 담당)
   • 공정:④ TTS 모델 구성 / ⑤ 음성 변환 로직 구성
   • 상세 업무:ElevenLabsAPI를 활용한 부모님 음성 클로닝. LLM에서 전달받은 감정 태그(기쁨, 슬픔 등)에 따라 일레븐랩스의 Stability, Similarity, Style Exaggeration파라미터를 동적으로 조절하는 변환 규칙 설계.
   • 핵심 도구:ElevenLabs API, Python

2. 콘텐츠 및 감정 설계 (NLP 담당)
   • 공정:① 전체 구조 설계 / ② 데이터 조사·수집
   • 상세 업무:동화 원문을 아이들이 이해하기 쉬운 구어체로 정제. **LLM(Llama 3/OpenAI)**을 활용해 문맥을 분석하고, 각 문장에 적합한 감정 라벨을 추출하는 프롬프트 엔지니어링 수행.
   • 핵심 도구:OpenAI API, LangChain, Prompt Engineering

3. 인터랙티브 대화 (STT 담당)
   • 공정:③ STT 기능 구현
   • 상세 업무:OpenAI Whisper를 활용해 아이의 음성을 실시간 텍스트로 변환. 아이가 말을 멈췄을 때를 감지하는 VAD(Voice Activity Detection) 로직을 구현하여 자연스러운 대화 흐름 생성.
   • 핵심 도구:Faster-Whisper, PyAudio

4. 백엔드 시스템 (API 담당)
   • 공정:⑥ 전체 통합·테스트
   • 상세 업무:FastAPI를 이용해 STT-LLM-TTS 모듈을 하나로 연결. 데이터가 각 모델 사이를 끊김 없이 흐르도록 비동기 파이프라인 설계 및 API 엔드포인트 관리.
   • 핵심 도구:FastAPI, Pydantic, SQLAlchemy

5. 사용자 화면 (UI/UX 담당)
   • 공정:⑦ 결과 데모 제작
   • 상세 업무:Streamlit을 활용하여 태블릿 전용 동화책 뷰어 제작. 아이가 직관적으로 조작할 수 있는 큰 마이크 버튼, 동화 삽화 슬라이드, 그리고 감정 변화를 시각적으로 보여주는 인터페이스 구현.
   • 핵심 도구:Streamlit, CSS, HTML
6. 인프라 및 통합 (DevOps 담당)
   • 공정:① 환경 구축 / ⑥ 전체 시스템 안정성 테스트
   • 상세 업무:DockerDocker-Compose를 사용하여 6명의 작업물을 컨테이너화. 모델 간 라이브러리 충돌을 방지하고, 태블릿에서 서버에 원활히 접속할 수 있도록 네트워크 및 외부 노출(ngrok 등) 환경 구축.
   • 핵심 도구:Docker, Docker-Compose, NVIDIA Docker
